2019-12-03: Backing Up Data Using FTP and Box

Have you heard of the 3-2-1 rule of thumb for backing up your data? You want to keep at least 3 copies of your data where 2 copies are backups on different storage media and 1 of the backups is located offsite.

Why is the 3-2-1 rule important, you ask? Losing months of work on a project is devastating. About 3 months before I defended my master's thesis, my laptop's graphics card died. The only way I could access my files was by booting my laptop in recovery mode. I only got some of them off and lost nearly every homework assignment, paper, and piece of code from my previous 5 years of classes. I even ended up losing over a decade's worth of photos. (Ironically, my phone also quit on me around the same time.) I was able to use backups of the documents and code I had on GitLab to finish my thesis, but I lost so many valuable resources in that crash.

Fast forward to right now, when I'm about 4 months away from defending my dissertation. I back my code and dissertation document up to Github and do most of my other work in Google Drive, but I am still worried about losing my data. It took months to get it all, and I am still running experiments using it. (Also, my phone is starting to show signs that its time is up.) While taking a quick break in my writing, I'm focusing on backing up my data. I'm in the process of getting an external hard drive with enough capacity to hold all of the data and results for my dissertation project, so I started with the offsite.

Through my grad program, I have unlimited data storage with Box. The biggest difficulty I've had so far with Box is that the drag-and-drop webpage interface is inefficient when uploading large amounts of data or a directory with multiple subdirectory levels. It seems like the GUI can't handle anything too complex. After doing a little digging, though, I learned that data could be transferred to and from Box using FTP.

Great! There are a number of file managers capable of FTP. I installed my choice, Filezilla, from Ubuntu Software, used the config parameters found here to set up my connection to Box (port 990, not 21), and clicked the "Quickconnect" button in Filezilla to start my connection. I was greeted with a 530 error - Invalid username or password. I retyped my password to be safe and tried again. No dice.

I started a chat with Tech Support at my school. We determined that my account had admin permission to use FTP on Box, that I was using the correct username/email address, and that my connection settings were correct. To be safe, I uninstalled Filezilla a la Ubuntu Software and installed it (and several dependencies) from the source code. This process was a little lengthy. I started up the newly installed Filezilla, retyped my settings, and tried to connect. Still no luck.

The Tech Support representative who had been helping me submitted an official ticket for me while I continued to look for a solution. Searching for the error code I had been receiving in Box's support community threads revealed the answer: FTP does not support single sign-on.

I had always accessed Box using my single sign-on username/password for school, but I had never set up an external password for Box. A quick trip to my Account Settings and one external password later, I am now able to connect to Box via FTP through Filezilla. Once my data finishes uploading, I will have my first complete backup.
